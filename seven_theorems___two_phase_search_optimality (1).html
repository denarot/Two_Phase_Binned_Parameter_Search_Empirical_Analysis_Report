Problem Setup

Given:
  • Random Forest performance function \(f(k)\) where \(k\) = number of trees
  • Search space \([k_{\min}, k_{\max}]\) with size \(N = k_{\max} - k_{\min}\)
  • Target: Find \(k^*\) such that \(f(k^*) \geq \max f(k) - \varepsilon\) (ε-optimal solution)
Assumptions:
  1. \(f(k)\) is monotonically increasing then plateaus: \(\exists k_p\) such that \(f'(k) \approx 0\) for \(k \geq k_p\)
  2. \(f(k)\) is Lipschitz continuous with constant \(L\)
  3. Evaluations have bounded noise: \(\tilde{f}(k) = f(k) + \eta\), where \(|\eta| \leq \sigma\)
Theorem 1: Phase 1 Exponential Search Complexity
Statement:
Let \(k^*\) denote the optimal tree count where the plateau begins. Phase 1 (exponential bracketing) finds an interval \([L, U]\) containing \(k^*\) in at most: $$\left\lceil \log_2\left(\frac{k^*}{k_{\min}}\right) \right\rceil + 1 \text{ evaluations}$$
Proof:
Phase 1 tests \(k_i = k_{\min} \cdot 2^i\) for \(i = 0, 1, 2, \ldots\)
  1. The search terminates when \(k_i \geq k^*\), which first occurs at \(i^* = \lceil\log_2(k^*/k_{\min})\rceil\)
  2. Number of evaluations = \(i^* + 1\) (includes \(i^*\) and bracketing check)
  3. Therefore: evaluations \(\leq \lceil\log_2(k^*/k_{\min})\rceil + 1 = O(\log(k^*/k_{\min}))\)
  4. Since \(k^* \leq k_{\max}\), this is bounded by \(O(\log(k_{\max}/k_{\min})) = O(\log N)\)
Complexity: O(log N) ∎
Theorem 2: Plateau Detection Correctness
Statement:
Given window size \(w \geq 3\) and threshold \(\varepsilon > 0\), the windowed gradient detection criterion: $$\frac{1}{w}\sum_{j=0}^{w-1} \left[f(k_{i+j+1}) - f(k_{i+j})\right] < \varepsilon$$ correctly identifies the plateau region with probability at least \(1 - \delta\), where: $$\delta \leq \exp\left(-\frac{w\varepsilon^2}{8\sigma^2}\right)$$
Proof:
  1. Let \(\Delta G_i = [f(k_{i+1}) - f(k_i)]\) be the gradient estimate at step \(i\)
  2. The windowed average is: \(\bar{G}_w = \frac{1}{w}\sum_{j=0}^{w-1} \Delta G_{i+j}\)
  3. If noise \(\eta \sim N(0, \sigma^2)\), then \(\bar{G}_w \sim N(\mu_{\text{true}}, \sigma^2/w)\) by CLT
  4. Apply Hoeffding's inequality for bounded random variables: $$P(|\bar{G}_w - \mu_{\text{true}}| > \varepsilon) \leq 2\exp\left(-\frac{2w\varepsilon^2}{(b-a)^2}\right)$$
  5. With bounded range \([a,b]\) and simplification for \(\sigma\) notation: $$P(\text{false detection}) \leq \exp\left(-\frac{w\varepsilon^2}{8\sigma^2}\right)$$
  6. Larger window \(w\) increases detection confidence exponentially
Robustness: Exponential decay in false positive rate ∎
Theorem 3: Phase 2 Binary Search Complexity
Statement:
Given bracketing interval \([L, U]\) from Phase 1 and precision requirement \(\delta > 0\), Phase 2 (binary refinement) finds \(k^*\) within \(\delta\) in: $$\left\lceil \log_2\left(\frac{U - L}{\delta}\right) \right\rceil \text{ evaluations}$$
Proof:
  1. Binary search halves the search interval each iteration
  2. After \(t\) iterations, interval width = \((U - L) / 2^t\)
  3. To achieve precision \(\delta\), we require: $$\frac{U - L}{2^t} \leq \delta \implies 2^t \geq \frac{U - L}{\delta}$$
  4. Taking \(\log_2\) of both sides: $$t \geq \log_2\left(\frac{U - L}{\delta}\right)$$
  5. Number of evaluations = \(\lceil\log_2((U-L)/\delta)\rceil = O(\log((U-L)/\delta))\)
  6. Since \(U - L \leq N\), this is \(O(\log(N/\delta)) = O(\log N + \log(1/\delta))\)
Complexity: O(log N + log(1/δ)) ∎
Theorem 4: Combined Two-Phase ε-Optimality
Statement:
The combined two-phase algorithm finds an ε-optimal solution \(\hat{k}\) such that: $$f(k^*) - f(\hat{k}) \leq \varepsilon$$ with total evaluations bounded by: $$\left\lceil \log_2\left(\frac{k^*}{k_{\min}}\right) \right\rceil + \left\lceil \log_2\left(\frac{k_{\max} - k_{\min}}{\delta}\right) \right\rceil + 2 = O(\log N + \log(1/\delta))$$
Proof:
  1. Phase 1 bound: From Theorem 1, Phase 1 uses \(\leq \lceil\log_2(k^*/k_{\min})\rceil + 1\) evaluations
  2. Phase 2 bound: From Theorem 3, Phase 2 uses \(\leq \lceil\log_2((U-L)/\delta)\rceil\) evaluations
  3. Total evaluations: $$T_{\text{total}} = \underbrace{\lceil \log_2(k^*/k_{\min}) \rceil + 1}_{\text{Phase 1}} + \underbrace{\lceil \log_2((U-L)/\delta) \rceil}_{\text{Phase 2}}$$
  4. Since \(U - L \leq k_{\max} - k_{\min} = N\) and \(k^* \leq k_{\max}\): $$T_{\text{total}} \leq \lceil \log_2(N) \rceil + \lceil \log_2(N/\delta) \rceil + 1$$ $$= \lceil \log_2(N) \rceil + \lceil \log_2(N) + \log_2(1/\delta) \rceil + 1$$ $$= O(\log N + \log(1/\delta))$$
  5. ε-Optimality: By Lipschitz continuity (constant \(L\)), if \(|\hat{k} - k^*| \leq \delta\): $$|f(\hat{k}) - f(k^*)| \leq L \cdot \delta$$ Choose \(\delta = \varepsilon/L\) to guarantee \(f(k^*) - f(\hat{k}) \leq \varepsilon\)
Complexity + Optimality: O(log N + log(1/δ)) evaluations, ε-optimal ∎
Theorem 5: Convergence Rate Guarantee
Statement:
Let \(k_t\) denote the midpoint estimate after \(t\) iterations of Phase 2 binary search with initial interval \([L, U]\). The error from optimum decreases geometrically: $$|k_t - k^*| \leq \frac{U - L}{2^t}$$ And by Lipschitz continuity (constant \(L\)): $$|f(k_t) - f(k^*)| \leq \frac{L(U - L)}{2^t}$$
Proof:
  1. Position error: Binary search maintains \(k^* \in [L_t, U_t]\) with width \((U_t - L_t) = (U - L)/2^t\)
  2. The midpoint \(k_t = (L_t + U_t)/2\) is at most \((U - L)/(2^{t+1})\) from \(k^*\)
  3. Bounding more conservatively: \(|k_t - k^*| \leq (U - L)/2^t\)
  4. Function error: By Lipschitz continuity: $$|f(k_t) - f(k^*)| \leq L \cdot |k_t - k^*| \leq \frac{L(U-L)}{2^t}$$
  5. To achieve ε accuracy: Solve for \(t\): $$\frac{L(U-L)}{2^t} \leq \varepsilon \implies t \geq \log_2\left(\frac{L(U-L)}{\varepsilon}\right)$$
Convergence: Exponential (geometric) rate of 1/2^t ∎
Theorem 6: Noise Robustness Under Stochastic Evaluations
Statement:
Let \(\tilde{f}(k) = f(k) + \eta\) where \(\eta \sim N(0, \sigma^2)\) represents evaluation noise. Using window size \(w\) for gradient estimation, the probability of false plateau detection is bounded by: $$P(\text{false plateau} \mid \text{true gradient} > 2\varepsilon) \leq \exp\left(-\frac{w\varepsilon^2}{2\sigma^2}\right)$$
Proof:
  1. True gradient: Let \(\mu = E[f(k_{i+1}) - f(k_i)] > 2\varepsilon\) (not at plateau)
  2. Observed gradient: With noise, observed differences are: $$\tilde{\Delta}_j = f(k_{i+j+1}) - f(k_{i+j}) + (\eta_{i+j+1} - \eta_{i+j})$$
  3. Windowed average: \(\bar{\Delta}_w = \frac{1}{w}\sum_{j=0}^{w-1} \tilde{\Delta}_j\)
  4. Expected value: \(E[\bar{\Delta}_w] = \mu > 2\varepsilon\)
  5. Variance: \(\text{Var}(\bar{\Delta}_w) = 2\sigma^2/w\) (independent noise cancels)
  6. False alarm: Occurs when \(\bar{\Delta}_w < \varepsilon\) despite \(\mu > 2\varepsilon\)
  7. Apply Hoeffding's bound: $$P(\bar{\Delta}_w < \varepsilon \mid \mu > 2\varepsilon) \leq P(|\bar{\Delta}_w - \mu| > \varepsilon)$$ $$\leq 2\exp\left(-\frac{2w\varepsilon^2}{4\sigma^2}\right) = 2\exp\left(-\frac{w\varepsilon^2}{2\sigma^2}\right)$$
  8. For practical purposes with large \(w\), bound simplifies to \(\exp(-w\varepsilon^2/(2\sigma^2))\)
Robustness: Exponentially decreasing false positive rate with window size ∎
Theorem 7: Information-Theoretic Lower Bound (Asymptotic Optimality)
Statement:
Any deterministic comparison-based algorithm that finds an ε-optimal solution \(k^*\) in a monotonic function \(f(k)\) over interval \([k_{\min}, k_{\max}]\) requires at least: $$\Omega\left(\log\left(\frac{k_{\max} - k_{\min}}{\varepsilon/L}\right)\right) = \Omega(\log N + \log(L/\varepsilon)) \text{ evaluations}$$ where \(L\) is the Lipschitz constant. Our two-phase algorithm achieves this bound, proving asymptotic optimality.
Proof:
  1. Information content: To specify \(k^*\) within precision \(\delta = \varepsilon/L\) in continuous space of size \(N\) requires: $$I = \log_2(N/\delta) = \log_2(N) + \log_2(L/\varepsilon) \text{ bits of information}$$
  2. Comparison model: Each function evaluation provides a comparison outcome (binary: better/worse), yielding at most 1 bit of information
  3. Lower bound: At least \(\lceil I \rceil = \Omega(\log N + \log(L/\varepsilon))\) evaluations required to extract \(I\) bits
  4. Adversarial argument: Consider worst-case input where \(k^*\) is positioned to maximize queries:
    • Binary search on \(N\) elements requires \(\lceil\log_2 N\rceil\) comparisons (proven optimal)
    • Precision requirement adds \(\log_2(1/\delta) = \log_2(L/\varepsilon)\) term
  5. Our algorithm: By Theorem 4, our method uses \(O(\log N + \log(1/\delta)) = O(\log N + \log(L/\varepsilon))\) evaluations
  6. Matching bounds: $$\Omega(\log N + \log(L/\varepsilon)) \leq T_{\text{our}} = O(\log N + \log(L/\varepsilon))$$
Conclusion: Our algorithm is asymptotically optimal (matches lower bound) ∎
